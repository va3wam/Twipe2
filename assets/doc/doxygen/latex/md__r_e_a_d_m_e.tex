The Arduino C++ code was written to run on the Espressif E\+SP W\+R\+O\+O\+M32 \char`\"{}\+Software On a Chip\char`\"{} (S\+OC). The development board used for this S\+OC is the Adafruit Huzzah32 featherboard. Note that some standard Arduino functions such as Analog\+Write() have not been implemented on the E\+S\+P32 S\+OC and alternate functions such as ledc\+Setup(), ledc\+Attach\+Pin() and ledc\+Write() must be used in their place. There are some additional compatibility issues of note as follows\+:


\begin{DoxyEnumerate}
\item The G\+P\+IO pin usage is loosey Adafruit featherboard compliant meaning it will work with some boards such as the O\+L\+ED board
\item This code does not use the Twipe pinout configuration or naming convension so will not port over to a Twipe compliant robot unmodified
\item This code will N\+OT run on an E\+S\+P8266 unmodified because of both G\+P\+IO incompatibility as well as some E\+S\+P32 specific libraries that are in use
\end{DoxyEnumerate}\hypertarget{md__r_e_a_d_m_e_autotoc_md32}{}\doxysubsection{Installing}\label{md__r_e_a_d_m_e_autotoc_md32}
