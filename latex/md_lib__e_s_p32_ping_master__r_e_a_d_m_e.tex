Let the ESP32\+Ping ping a remote machine.

\#\+Note that this is a port from \href{https://github.com/dancol90/ESP8266Ping}{\texttt{ https\+://github.\+com/dancol90/\+ESP8266\+Ping}} With this library an ESP32\+Ping can ping a remote machine and know if it\textquotesingle{}s reachable. It provide some basic measurements on ping messages (avg response time).\hypertarget{md_lib__e_s_p32_ping_master__r_e_a_d_m_e_autotoc_md47}{}\doxysection{Usage}\label{md_lib__e_s_p32_ping_master__r_e_a_d_m_e_autotoc_md47}
First, include the library in your sketch along with Wi\+Fi library\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\#include <WiFi.h>}
\DoxyCodeLine{\#include <ESP32Ping.h>}

\end{DoxyCode}


Next, simply call the {\ttfamily Ping.\+ping()} function


\begin{DoxyCode}{0}
\DoxyCodeLine{IPAddress ip (192, 168, 0, 1); // The remote ip to ping}
\DoxyCodeLine{bool ret = Ping.ping(ip);}

\end{DoxyCode}


{\ttfamily ret} will be true if the remote responded to pings, false if not reachable. The library supports hostname too, just pass a string instead of the ip address\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{bool ret = Ping.ping("{}www.google.com"{});}

\end{DoxyCode}


Additionally, the function accept a second integer parameter {\ttfamily count} that specify how many pings has to be sent\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{bool ret = Ping.ping(ip\_or\_host, 10);}

\end{DoxyCode}


After {\ttfamily Ping.\+ping()} has been called, the average response time (in milliseconds) can be retrieved with


\begin{DoxyCode}{0}
\DoxyCodeLine{float avg\_time\_ms = Ping.averageTime();}

\end{DoxyCode}
